# Chet.Template.GUI框架升级和重构计划

## 项目现状
- 解决方案包含三个项目：
  - Chet.Template.GUI（主WPF应用程序，net5.0-windows）
  - Chet.Template.GUI.ApplicationService（应用服务层，net5.0）
  - Chet.Template.GUI.Domain（领域层，net5.0）
- 主要依赖：
  - Volo.Abp.Autofac（4.2.2）- ABP框架的Autofac集成
  - Volo.Abp（4.2.2）- ABP框架核心
  - Octokit（0.50.0）- GitHub API客户端
  - Microsoft.Extensions.Hosting（5.0.*）- .NET主机服务
  - Serilog系列包 - 日志框架

## 升级和重构目标
1. 将框架升级到.NET 8
2. 移除ABP相关框架
3. 使用.NET 8内置功能重构

## 实施步骤

### 1. 升级项目框架版本
- 修改所有项目的TargetFramework：
  - Chet.Template.GUI：net5.0-windows → net8.0-windows
  - Chet.Template.GUI.ApplicationService：net5.0 → net8.0
  - Chet.Template.GUI.Domain：net5.0 → net8.0

### 2. 移除ABP相关依赖
- 从各项目中移除ABP相关的NuGet包：
  - Volo.Abp
  - Volo.Abp.Autofac
  - Volo.Abp.Castle.Core（如存在）
  - 其他ABP相关包

### 3. 重构依赖注入机制
- 将ABP的依赖注入替换为.NET内置的DI容器
- 移除ABP的依赖注入特性（如ITransientDependency、ISingletonDependency等）
- 使用Microsoft.Extensions.DependencyInjection的注册方式

### 4. 重构模块系统
- 移除ABP的模块系统（AbpModule、DependsOn等）
- 重构模块初始化逻辑，使用.NET 8的主机服务初始化
- 移除所有Module类（TemplateGUIDomainModule、TemplateGUIApplicationService、GUIModule）

### 5. 更新其他依赖包
- 更新Microsoft.Extensions.Hosting到兼容.NET 8的版本
- 更新Serilog系列包到兼容.NET 8的版本
- 更新Octokit到兼容.NET 8的版本

### 6. 重构代码
- 重构App.xaml.cs，移除ABP应用程序初始化逻辑
- 重构服务类，移除ABP相关特性
- 重构MainWindow.xaml.cs，简化依赖注入
- 更新配置管理，使用.NET 8的配置系统

### 7. 测试和验证
- 确保应用程序能够正常编译
- 确保应用程序能够正常运行
- 验证核心功能（模板下载、解压、替换）正常工作
